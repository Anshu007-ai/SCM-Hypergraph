1. Introduction & Motivation

Modern supply chains are complex, multi-party networks where a single disruption — a port closure, a supplier bankruptcy, a raw material shortage — can cascade through hundreds of interconnected entities causing billions in losses. Traditional graph neural networks model relationships as pairwise edges (A → B), but supply chain interactions are inherently multi-way: a single transaction involves a supplier, a manufacturer, a logistics provider, and a quality inspector simultaneously.

Hypergraphs solve this by allowing a single edge (called a hyperedge) to connect any number of nodes at once. This is the core insight of HT-HGNN: model supply chain relationships as hyperedges rather than simple edges.

Why Hypergraphs? In a regular graph: Edge = {Supplier A → Factory B}. Only 2 entities per edge.
In a hypergraph: Hyperedge = {Supplier A, Supplier B, Part X, Factory C, Logistics D}. Any number of entities per edge.
This captures the reality that supply chain events involve groups of entities acting together.

HT-HGNN v2.0 adds three further capabilities beyond basic hypergraph modeling:

  • Temporal awareness: Supply chain risk changes over time (seasonal demand, delivery delays, market shifts). A Bi-LSTM + Transformer fusion encoder captures both short-term and long-range temporal patterns.
  • Heterogeneous relations: Not all connections are the same. A "supplier_of" relationship is fundamentally different from "transported_by" or "co_disrupted_with". The model learns separate attention weights for each of the 5 relation types.
  • Cascade simulation: When one node fails, risk propagates through connected hyperedges. The cascade risk head simulates this propagation over 12 time steps, predicting how far and how fast a disruption will spread.

4. Phase 1 — Data Ingestion & Cleaning

The pipeline accepts any CSV file containing supply chain data. The system automatically converts it into the required numeric format through the following steps:

4.1 CSV Parsing

The uploaded CSV is read using pandas.read_csv(). The system does not require any specific column naming convention — it auto-detects the structure.

4.2 Node Name Extraction

The system scans columns left-to-right looking for the first non-numeric column (dtype = object/string). This column is extracted as node names (e.g., supplier names, part IDs, factory codes). This allows the final output to show meaningful labels like "Supplier_A" instead of "node_0".

4.3 Numeric Conversion

All remaining columns are converted to numeric values using pd.to_numeric(errors='coerce'). Any value that cannot be parsed as a number (text, dates, special characters) becomes NaN.

4.4 Cleaning

  • Drop all-NaN rows: Rows with no numeric data at all are removed
  • Drop all-NaN columns: Columns that were entirely text (now all NaN) are removed
  • Fill remaining NaN with 0: Sparse missing values are filled with zero — the model can handle this
Key Takeaway: Users can upload any CSV with mixed text and numeric columns. The system automatically strips text, fills gaps, and produces a clean N×F numeric matrix (N rows/nodes, F feature columns). No manual pre-formatting needed.

5. Phase 2 — Feature Normalization

Raw features have wildly different scales — a delivery time might be 1–30 days while a cost is 100–50,000 dollars. Without normalization, the neural network would be dominated by high-magnitude features.

5.1 Min-Max Scaling

First, all features are scaled to the [0, 1] range:

X_norm = (X − X_min) / (X_max − X_min)

This ensures every feature has the same numeric range, so no single feature dominates due to its natural scale.

5.2 Z-Score Normalization

Then, features are centered around zero with unit variance:

X_standardized = (X − μ) / σ

where μ is the mean and σ is the standard deviation. This helps gradient-based optimizers converge faster because the loss surface becomes more symmetric.

6. Phase 3 — Hypergraph Construction

This is the core innovation of HT-HGNN. Instead of building a simple graph with pairwise edges, we build a hypergraph where each edge can connect any number of nodes simultaneously.

6.1 Nodes (V)

Each entity in the supply chain becomes a node. Node types include:

  • Suppliers — raw material or component providers
  • Parts/Components — manufactured items in the bill of materials
  • Factories/Manufacturing steps — assembly and production entities
  • Logistics nodes — ports, warehouses, transport links

Each node carries an F-dimensional feature vector (e.g., reliability score, lead time, cost, substitutability, location risk).

6.2 Hyperedges (E)

A hyperedge groups nodes that participate in the same supply chain interaction. Examples:

Hyperedge TypeExampleNodes Involved
Supply TransactionOrder #4521{Supplier A, Part X, Factory C, Shipper D}
BOM AssemblyEngine Block{Part_1, Part_2, Part_3, Assembly_Line_A}
Co-Disruption EventPort Congestion{Port_SH, Port_SZ, Port_NB, Carrier_MSK}
Quality ChainCert #789{Supplier B, Inspector Q, Factory C}

6.3 Incidence Matrix (H)

The mathematical representation of the hypergraph is an incidence matrix H of size N × M (N nodes, M hyperedges):

H[v, e] = 1    if node v belongs to hyperedge e
H[v, e] = 0    otherwise

This sparse binary matrix is the primary data structure that flows through all subsequent neural network layers.

6.4 Dynamic Hyperedge Mining

Beyond static relationships, the DynamicHyperedgeConstructor discovers hidden patterns by analyzing temporal co-occurrences. If nodes are repeatedly disrupted together within sliding time windows, they form a new "co_disrupted_with" hyperedge. Similar hyperedges are merged using Jaccard similarity (threshold 0.7).

7. Phase 4 — Risk Label Generation (HCI)

Before training, we need ground-truth labels — how risky is each part of the supply chain? The Hypergraph Critical Index (HCI) is computed from the hypergraph structure itself:

7.1 Joint Failure Probability

P(joint failure) = 1 − ∏v ∈ e reliability(v)

For each hyperedge, multiply the reliability scores of all member nodes. The complement gives the probability that at least one node in the group will fail. If any node has low reliability, the entire group is at risk.

7.2 Engineering Impact

Impact = ∛(BOM_weight × tolerance × (1 + critical_path))

Combines three factors: how important the component is in the bill of materials (BOM weight), manufacturing precision requirements (tolerance), and whether it's on the critical path (binary flag). The cube root prevents any single factor from dominating.

7.3 Propagation Risk

Propagation = |downstream_hyperedges| / |total_hyperedges|

Measures what fraction of the entire supply chain would be affected if this hyperedge fails. Hyperedges that feed into many downstream processes have high propagation risk.

7.4 Supplier Concentration Risk

Concentration = (low_suppliers + 0.5 × substitution_risk + 0.3 × lead_time_risk) / 2.3

Captures single-point-of-failure risk: few available suppliers, low substitutability, and long lead times all increase concentration risk.

7.5 Final HCI Score

HCI = α × P(joint_failure) + β × Engineering_Impact + γ × Propagation_Risk

where α=0.5, β=0.3, γ=0.2

Final_Risk = 0.7 × HCI + 0.3 × Concentration_Risk
Why This Matters: The HCI score combines structural (who's connected to whom), reliability (how likely to fail), and impact (how bad if it does fail) into a single [0,1] score. This is what the neural network learns to predict — and then generalize to unseen data.

8. Phase 5 — Neural Network Architecture

The trained model consists of 4 sequential layers, each adding a different type of intelligence. The input is the node feature matrix X (N×F) and incidence matrix H (N×M). The output is a rich embedding for each node.

8a. Layer 1 — Spectral Hypergraph Convolution

This layer enables information flow through multi-way relationships. Unlike standard GCN which only passes messages between pairs, this operates on the full hypergraph.

Step 1: Node → Hyperedge Aggregation

hyperedge_features = HT × X     (M×N) × (N×F) = (M×F)

Each hyperedge collects features from all its member nodes by matrix multiplication. This produces a feature vector for each hyperedge that represents the "group signal".

Step 2: Attention Weighting

attention = softmax(learnable_weights)    → per-hyperedge importance

A learnable attention vector assigns importance weights to each hyperedge. The softmax ensures weights sum to 1. The model learns which groups matter most.

Step 3: Hyperedge → Node Propagation

X_out = H × (attention ⊙ hyperedge_features)     (N×M) × (M×F) = (N×F)

The weighted hyperedge signals are propagated back to nodes. Each node receives aggregated information from all the groups it belongs to. A residual connection adds the original features.

Intuition: Imagine a supplier belongs to 3 different transaction groups. After this layer, that supplier's embedding contains information from all 3 groups — it "knows" about the other suppliers, parts, and factories it regularly interacts with.

8b. Layer 2 — Heterogeneous Graph Transformer (HGT)

Not all relationships are equal. A "supplier_of" connection carries different information than "transported_by" or "co_disrupted_with". The HGT layer processes each relationship type separately.

5 Relation Types:

RelationMeaningExample
supplier_ofProvides materials/partsFoxconn → iPhone display
manufactured_byAssembled/produced atEngine → Pune Factory
transported_byShipped via logisticsCargo → Maersk shipping
quality_controlled_byInspected/certifiedPart → DGCA inspector
co_disrupted_withHistorically fail togetherPort Shanghai ↔ Port Shenzhen

Processing:

  1. Type-specific projection: Each node type (supplier/part/transaction) gets its own linear transformation Wtype
  2. Per-relation transformer: Multi-head self-attention is applied independently for each of the 5 relation types
  3. Attention fusion: Outputs from all 5 relations are concatenated and fused through a linear layer with softmax attention

8c. Layer 3 — Temporal Fusion Encoder

Supply chain risk is time-dependent. Seasonal demand peaks, delivery pattern changes, and post-disruption recovery all create temporal signals. This layer combines two complementary temporal models:

Bi-LSTM (Short-term patterns)

A bidirectional LSTM reads the node feature sequence forward and backward in time. It excels at capturing short-term temporal patterns like seasonal demand spikes, weekly delivery cycles, and immediate post-disruption effects.

Transformer (Long-range dependencies)

A multi-head self-attention Transformer captures long-range temporal dependencies — patterns that span months or years, such as annual supply cycles, gradual supplier degradation, or slow-moving market shifts.

Learned Gating

output = α × LSTM_output + (1 − α) × Transformer_output

where α = sigmoid(learnable_parameter)

Rather than manually weighting the two temporal signals, the model learns the optimal blend. During training, α adapts — for fast-changing supply chains it may favor LSTM; for stable long-term patterns it may favor the Transformer.

8d. Layer 4 — Cascade Risk Head

The most novel component. When one node is disrupted, risk cascades through the hypergraph — if a key supplier fails, all hyperedges containing that supplier are weakened, which in turn threatens other nodes in those groups.

Cascade Propagation Algorithm (12 steps):

  1. Initial estimate: An MLP maps each node's embedding to a disruption probability P(v) ∈ [0, 1]
  2. Hyperedge risk: For each hyperedge, compute the probability that the group is disrupted:
    P(e disrupted) = 1 − ∏v ∈ e (1 − P(v))α

    where α is a learnable interaction strength parameter

  3. Node risk update: Each node takes the maximum risk across all its hyperedges:
    P(v) = maxe ∋ v P(e disrupted)
  4. Damping: Apply factor β ≈ 0.9 to prevent unbounded amplification:
    P(v) = P(v) × β
  5. Repeat steps 2–4 for 12 time steps, simulating how disruption waves spread through the network
Example: If Supplier A (P=0.8) belongs to Hyperedge E1 with Parts B, C, D — the group risk becomes 1 − (1−0.8)^α × (1−P(B))^α × ... This cascades outward through other hyperedges containing B, C, or D, mimicking real-world domino effects.

The final node embeddings from the 4-layer network are fed into 4 parallel prediction heads, each specialized for a different task:

10. Phase 7 — Training & Optimization

10.1 Loss Function

L_total = 1.0 × L_price + 0.8 × L_change + 1.2 × L_criticality + 0.6 × L_cascade

Criticality gets the highest weight (1.2) because accurate risk classification is the primary objective. Cascade risk gets the lowest (0.6) as it's a secondary, harder-to-validate signal.

10.2 Optimization

  • Optimizer: Adam with adaptive learning rate (initial LR = 0.001)
  • Scheduler: ReduceLROnPlateau — halves LR when validation loss stagnates
  • Epochs: 100 (with early stopping if no improvement for 15 epochs)
  • Gradient clipping: Max norm = 1.0 (prevents exploding gradients)

10.3 Backpropagation

The total loss is differentiated with respect to all model parameters using PyTorch's automatic differentiation. Gradients flow backward through all 4 layers — from prediction heads → cascade head → temporal encoder → HGT → hypergraph conv → input projections. The Adam optimizer updates every weight to minimize the total loss.

11. Phase 8 — Output, Explainability & Simulation

11.1 Predictions

For each node in the supply chain, the model outputs:

  • Price prediction: Expected price/cost of the component
  • Change forecast: Expected percentage change (−100% to +100%)
  • Criticality score: Risk level from 0 (safe) to 1 (critical)
  • Cascade risk: Probability of being affected by cascading disruption

11.2 HyperSHAP Explainability

HyperSHAP extends Shapley value attribution to hypergraphs. For each prediction, it answers:

  • Which hyperedges most influenced the prediction? (e.g., "Hyperedge E_23 containing {Supplier X, Part Y} increased risk by 0.34")
  • Which features drove the score? (e.g., "lead_time had the highest attribution at 0.42")
  • Actionable recommendations: "Diversify suppliers for Part Y" or "Reduce lead time below 14 days"

The Shapley value for hyperedge e is computed as:

Φ(e) = ΣS ⊆ E\{e} [|S|! × (|E|−|S|−1)! / |E|!] × [f(S ∪ {e}) − f(S)]

where f(S) is the model's prediction when only the hyperedges in coalition S are active. Monte Carlo sampling (100 samples) makes this tractable.

11.3 Cascade Simulation

The cascade engine provides what-if scenario testing:

  1. User specifies a shock node (e.g., "What if Port Shanghai shuts down?")
  2. The engine runs the Independent Cascade model on the hypergraph for N time steps
  3. At each step, it tracks: which nodes are disrupted, which are newly at risk, and the overall spread percentage
  4. Output includes: cascade timeline, maximum spread, depth, and estimated recovery time
